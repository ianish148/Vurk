# Database Schema

The Vurk database is heavily optimized for a scalable, event-driven SaaS architecture.

## Key Principles
1. **UUID Primary Keys:** Every table uses UUIDs (`gen_random_uuid()`).
2. **Soft Deletes:** `deleted_at` timestamp on all entities instead of permanent deletions.
3. **Audit Trails:** Critical tables have `created_at` and `updated_at` (via triggers).
4. **Row Level Security (RLS):** All tables have strict RLS policies to prevent data leakage.
5. **Full-Text Search:** Postgres `tsvector` columns (`search_vector`) are pre-computed for fast querying.

## Core Schema Modules

### Auth & Profiles
* `profiles`: Extends Supabase Auth with custom user fields, streaks, and total XP.
* `user_preferences`: Separated from profile for better maintainability.

### The Roadmap Hierarchy
* `roadmap_templates` -> `roadmap_phases` -> `roadmap_milestones` -> `roadmap_modules` -> `roadmap_tasks`
* `roadmap_prerequisites`: Maps dependencies between templates.
* `task_dependencies`: Maps dependencies between tasks.

### User Engine
* `user_roadmaps`: Represents a user's active enrollment (subscription) to a template.
* `user_tasks`: Represents the materialized tasks generated by the Smart Scheduler.

### Gamification & Analytics
* `xp_transactions`, `coin_transactions`: Ledger-based economy tracking.
* `events`: Event log for triggering webhooks and asynchronous jobs.
* `ai_usage_logs`: Tracks token cost and success of AI verification.
